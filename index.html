Typography

The typography system is built around the Outfit variable font, providing a modern, readable base for all text. Body content uses a consistent font size and comfortable line height for optimal readability, while headings follow a clear scale with bold weight and tighter line spacing for strong visual hierarchy. Semantic elements share a unified text color, lists use intuitive inside markers with nested styles, and utility classes enable simple text alignment. Monospace fonts are reserved for code to ensure clarity and distinction.

Accordion

The Accordion component creates collapsible content sections. Uses hidden radio/checkbox inputs with labels and is CSS only

This is a pure CSS accordion component. It uses the "Radio Button Hack" to manage state without a single line of JavaScript.

It links label elements to hidden radio buttons.

It's reasonably accessible as it uses semantic form elements. Users can tab through the headers (radio inputs) and select them with the keyboard. However, for full ARIA support, a small amount of JS is usually recommended to manage aria-expanded attributes.

The DatePicker component provides a calendar interface for date selection. Supports single date or date range modes, customizable locales, and mobile-responsive design.

Parameter Type Description
input HTMLInputElement | string the input element to attach the date picker to
options DatePickerOptions an object with various configuration options for the date picker (see options table)
currentDate Date the current date to be displayed in the date picker; defaults to today's date
selectedDate Date the selected date in the date picker; defaults to null

    Gallery

    A responsive, infinite-scroll masonry gallery that dynamically arranges image cards into columns. The layout automatically adapts to the viewport width, redistributing items into the shortest column for a balanced, Pinterest-style grid. Cards feature lazy-loaded images with a smooth fade-in effect, titles, and descriptions. Scroll to the bottom to load more content fetching can be throttled with a configurable reload limit to prevent runaway requests.

    Constructor Parameters

    Parameter Type Description
    containerId string The id of the HTML element that will serve as the gallery container. Throws an error if not found in the DOM.
    options MasonryGalleryOptions Optional configuration object to customise the gallery's layout and behaviour. Defaults to {}.

    MasonryGalleryOptions

    Option Type Default Description
    minColumnWidth number 250 Minimum width (in pixels) for each masonry column. The number of columns is calculated by dividing the available container width by this value.
    scrollThreshold number 100 Distance from the bottom of the page (in pixels) at which the next batch of images is fetched. A lower value triggers loading later; a higher value triggers it earlier.
    loaderSelector string '.loader' CSS selector for the loading-indicator element. The element is shown/hidden automatically during fetch cycles via a hidden class toggle.
    reload number 2 Maximum number of times new images can be fetched via infinite scroll. Once the limit is reached, further scroll events are ignored.
    fetchFunction Promise<ImageData[]> — A promise that resolves to an array of ImageData objects ({ src, title, desc }). Provides the image data for each load cycle. Required in practice — the built-in fallback throws an error.
