Basix Component Demos

This page demonstrates all available components in the Basix library, including both TypeScript components and CSS-only components.

CSS-Only Components

These components require no JavaScript - just HTML markup with the appropriate CSS classes.

Alert

The Alert component displays contextual feedback messages. Available variants: default, .alert-success, .alert-warning, and .alert-error. Pure CSS - just add the class to any element.

Button

The Button component provides styled buttons with variants. Use <button> or .button class. Variants: .button-primary, .button-success, .button-warning, .button-error, and .active state.

Card

The Card component creates contained content sections with optional header and footer. Use .card class with optional .header and .footer divs inside.

Chips

The Chips component displays small interactive elements like tags or filters. Use .chips container with .chip items. Add .clickable for hover effects or .closeable for a close button.

Checkbox

The Checkbox component provides custom-styled checkboxes. Use input.styled-checkbox followed by a <label>. Supports :checked and :disabled states.

Radio Button

The Radio Button component provides custom-styled radio inputs. Wrap each radio in a .radio-button-container label with a .checkmark span.

Switch

The Switch component creates iOS-style toggle switches. Wrap a checkbox input and label in a .switch container.

Spinner

The Spinner component shows loading indicators. Use .spinner for a rotating ring or .loading for a three-dot pulsing animation.

Progress Bar

The Progress Bar component displays task completion. Use .progress-bar container with a .progress div inside. Set width via inline style or class.

Tooltip

The Tooltip component shows contextual information on hover. Add .tooltip class and use the title attribute for tooltip text.

The Accordion component creates collapsible content sections. Uses hidden radio/checkbox inputs with labels. Structure: .accordion > .accordion-item > .accordion-input + .accordion-label + .accordion-content.

The Timeline component displays chronological events. Use .timeline list with .timeline-item items containing .timeline-content, .timeline-date, .timeline-title, and .timeline-body.

Placeholder

The Placeholder component creates skeleton loading states. Use .placeholder with width classes .w-1 through .w-12 (12-column grid). Animates with a pulsing fade effect.

TypeScript Components

These components require JavaScript initialization for full functionality.

Tabs

The Tabs component creates accessible tabbed interfaces. Supports horizontal/vertical layouts, keyboard navigation (arrow keys, Home, End), and ARIA attributes. Initialize with new Tabs(selector, options).

Table

The Table component provides sortable, searchable, and paginated data tables. It can parse existing HTML tables or accept data programmatically. Initialize with new Table(selector, { data, columns, pageSize }).

Toast

The Toast component displays non-blocking notification messages. Supports types (success, error, warning, info), auto-dismiss with progress bar, and optional close button. Create with new Toast({ content, header, type, closeable }) then call .show(ms).

Modal

The Modal component creates dialog overlays with header, content, and footer sections. Supports types (success, error, warning, info), close on Escape key, and backdrop click. Initialize with new Modal({ content, header, footer, type }) then call .show().

Select

The Select component transforms native <select> elements into custom styled dropdowns. Supports single and multi-select modes. Wrap your select in a .select-group and initialize with new Select(selector).

DatePicker

The DatePicker component provides a calendar interface for date selection. Supports single date or date range modes, customizable locales, and mobile-responsive design. Initialize with new DatePicker(inputSelector, { mode, locales, onSelect }).

Carousel

The Carousel component creates image/content sliders with navigation buttons and dot indicators. Supports loop mode, autoplay, and touch/swipe gestures. Initialize with new Carousel(selector, { loop, autoPlay, autoPlayInterval }).

Tree

The TreeComponent renders hierarchical data as an expandable/collapsible tree. Supports file/folder icons, selection, and programmatic expand/collapse. Create nodes with new TreeNode(label, type, children) and initialize with new TreeComponent(selector, nodes).

    CodeViewer

    The CodeViewer component displays syntax-highlighted code with a copy button. Supports JavaScript, HTML, and CSS highlighting. Initialize with new CodeViewer(selector, codeString, language).

    FileUploader

    The FileUploader component provides drag-and-drop file upload functionality with progress indication. Supports file validation (size, type), multiple files, and upload cancellation. Requires specific HTML structure with #drop-zone, #file-input, #file-list, and #upload-btn.

    FlyoutMenu

    Theme

    The Theme component manages light/dark mode switching. Persists preference to localStorage, respects system preference, and supports keyboard shortcut (Ctrl/Cmd+J). Requires elements with IDs #theme-toggle and #theme-icon. Initialize with Theme.init().

    PushMenu

    The PushMenu component creates a sidebar navigation that "pushes" the main content when opened. Uses a checkbox-based toggle mechanism. Requires elements with classes .navigation, .push-content, .push-menu. Initialize with PushMenu.init().

    Note: PushMenu requires specific page layout structure and is best demonstrated in a full-page context.

    Editor

    The Editor component is a rich text (WYSIWYG) editor with formatting toolbar. Supports bold, italic, underline, links, images, lists, undo/redo, code view, and HTML export. Requires elements with IDs #editable, #code, #preview. Initialize with new Editor().

    Note: Editor requires specific toolbar and content structure. See the source for full implementation details.
