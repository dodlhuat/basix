Stepper

Multi-step process indicator with horizontal and vertical layouts. Steps can be completed, active, or in an error state. The Stepper class manages transitions programmatically.

Usage

Options

OptionTypeDefaultDescription
defaultStepnumber0Initial active step index.
clickablebooleanfalseAllows clicking completed or active steps to navigate.
iconBasePathstring'svg-icons/'Base path to the SVG sprite directory.
onChange(current, previous) => void—Callback fired when the active step changes.

Step states

ClassDescription
.completedStep has been successfully completed. The indicator shows a checkmark.
.activeCurrent active step.
.errorStep encountered a validation error. Re-apply after navigation if still needed.
.stepper-verticalSwitches to a vertical layout with description support.

API

MethodDescription
next()Marks the current step as completed and advances to the next.
prev()Returns to the previous step.
goTo(index)Jumps directly to a step by 0-based index.
getStep()Returns the current step index.
isFirst()Returns true if on the first step.
isLast()Returns true if on the last step.
setError(index)Applies the error state to the specified step.
clearError(index)Removes the error state from the specified step.
destroy()Removes event listeners and injected connector elements.
